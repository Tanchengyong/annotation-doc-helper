🚀 1. 定义与作用

技术定义：

@Documented 是 Java 语言中的元注解，用于指定被修饰的注解会被 JavaDoc 等工具提取到文档中。

核心作用：

  • 文档生成：让注解信息出现在 JavaDoc 文档中，方便开发者理解注解用途。
  • 增强可读性：为使用注解的类、方法等元素提供更详细的说明。

📌 2. 所属类别

类型： 元注解 (Meta Annotation)

关联机制：


        JavaDoc 工具
               ↓
        注解信息提取

通过 JavaDoc 工具将注解信息包含在生成的文档里。

🌍 3. 使用位置

代码示例：


        // 声明自定义注解并使用 @Documented
        @Documented
        @Target(ElementType.METHOD)
        @Retention(RetentionPolicy.RUNTIME)
        public @interface MyMethodAnnotation {
            String description() default "";
        }

        // 使用自定义注解
        public class MyClass {
            /**
             * 这是一个示例方法
             */
            @MyMethodAnnotation(description = "此方法用于执行特定任务")
            public void myMethod() {
                // 方法实现
            }
        }

应用规范：

  • 仅能用于注解声明的上方。
  • 与 @Target、@Retention 等元注解配合使用来定义注解的完整特性。

⚙ 4. 工作原理

  1. 注解声明阶段
    • 使用 @Documented 修饰自定义注解。
  2. 文档生成阶段
    • 当使用 JavaDoc 工具生成文档时，工具会识别带有 @Documented 的注解。
    • 将注解信息添加到对应元素（类、方法等）的文档中。

💡 5. 使用场景

场景解决方案技术实现
库开发 让库中的注解信息在文档中可见，方便使用者了解注解用途。 
@Documented @ApiVersion
框架开发 在框架文档中展示注解信息，帮助开发者更好地使用框架。 
@Documented @ComponentScan

💻 7. 完整示例


        // 声明可文档化的注解
        @Documented
        @Target(ElementType.TYPE)
        @Retention(RetentionPolicy.RUNTIME)
        public @interface MyClassAnnotation {
            String author() default "";
        }

        /**
         * 这是一个示例类
         */
        @MyClassAnnotation(author = "John Doe")
        public class MySampleClass {
            // 类实现
        }

        // 生成 JavaDoc 文档后，文档中会包含 MyClassAnnotation 的信息

运行效果：

  • 使用 JavaDoc 工具生成文档时，MySampleClass 的文档中会显示 @MyClassAnnotation 的信息，如作者信息。
